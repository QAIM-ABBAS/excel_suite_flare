# Cloudflare Pages + Workers Deployment Plan

## Overview

Deploy frontend to **Cloudflare Pages** and backend API to **Cloudflare Workers** with D1 database and R2 storage.

---

## Implementation Steps

### Phase 1: Frontend Static Export

**Goal:** Export Next.js as static files for Cloudflare Pages

1. Update `next.config.mjs` for static export
2. Remove server-side dependencies from frontend code
3. Update API calls to point to Worker URL
4. Test static build

### Phase 2: Backend Worker Setup

**Goal:** Create Cloudflare Worker with Hono framework

1. Initialize Worker project with `wrangler.toml`
2. Install Hono (lightweight framework for Workers)
3. Set up D1 database binding
4. Set up R2 bucket binding
5. Migrate API routes to Worker

### Phase 3: Database Migration

**Goal:** Replace SQLite with Cloudflare D1

1. Create D1 database schema
2. Update Prisma schema for D1 (or use Drizzle ORM)
3. Migrate database operations

### Phase 4: File Storage Migration

**Goal:** Replace filesystem with Cloudflare R2

1. Create R2 bucket
2. Update file upload to use R2
3. Update file download to use R2 presigned URLs
4. Handle file processing in-memory or with R2

**File:** `src/lib/api.ts`

Update to point to Worker URL:

```typescript
const API_BASE = process.env.NEXT_PUBLIC_API_URL || 'https://your-worker.workers.dev';

export async function apiFetch(path: string, init?: RequestInit): Promise<Response> {
  return fetch(`${API_BASE}${path}`, init);
}

export function downloadUrl(path: string): string {
  return `${API_BASE}${path}`;
}
```

### 5. R2 Storage Operations

```typescript
// worker/src/lib/r2.ts
export async function uploadToR2(
  bucket: R2Bucket,
  key: string,
  data: ArrayBuffer,
  contentType: string
): Promise<void> {
  await bucket.put(key, data, { httpMetadata: { contentType } });
}

export async function downloadFromR2(
  bucket: R2Bucket,
  key: string
): Promise<R2ObjectBody | null> {
  return await bucket.get(key);
}

export async function getPresignedUrl(
  bucket: R2Bucket,
  key: string,
  expiresIn: number = 3600
): Promise<string> {
  const url = await bucket.presignedUrl(key, { expiresIn });
  return url.toString();
}
```

### 6. Excel Processing (In-Memory)

Since Workers don't have filesystem, process Excel files in-memory:

```typescript
// worker/src/lib/excel.ts
import * as XLSX from 'xlsx';
import Papa from 'papaparse';

export async function processExcelInMemory(
  fileBuffer: ArrayBuffer,
  options: ProcessOptions
): Promise<ProcessResult> {
  // Read file from buffer
  const wb = XLSX.read(fileBuffer, { type: 'array' });
  
  // Process in memory
  const result = processWorkbook(wb, options);
  
  // Generate output buffer
  const outputBuffer = XLSX.write(result.workbook, { 
    type: 'array', 
    bookType: 'xlsx' 
  });
  
  return {
    buffer: outputBuffer,
    metadata: result.metadata
  };
}
```

## Deployment Commands

### Frontend (Cloudflare Pages)
```bash
# Build static export
npm run build

# Deploy to Cloudflare Pages
npx wrangler pages deploy out/
```

### Backend (Cloudflare Workers)
```bash
# Create D1 database
npx wrangler d1 create excel-suite-db

# Create R2 bucket
npx wrangler r2 bucket create excel-suite-files

# Initialize database
npx wrangler d1 execute excel-suite-db --file=schema.sql

# Deploy worker
npx wrangler deploy
```

| Cloudflare Pages | 500 builds/month, unlimited requests | $0 (free) |

**Total for moderate usage:** ~$5-10/month
